## Commit Conventions

### Commit syntax: 

<type>[scope]: <description>
[body]
[footer]

<> = required
[] = optional

### Commit types:

feat: for adding features
style: for style modifications
fix: for bug fixing
docs: for adding documentation
refactor: for code refactoring and optimisation
